## ✔ 퀘스트 1. 세 번 말하면 한 번 쉬기

### 💭 배경

공부를 하다 보면 집중해서 연속적으로 질문을 많이 하게 된다. 하지만 너무 몰입하면 금방 지치고 효율이 떨어질 수 있다.

### 🎯 목적

50분 공부하고 10분 쉬는 것 처럼, AI에게도 쉴 시간을 주자. AI가 쉴 때 나도 쉬어서 나에게도 휴식을 주자!

### 📝 진행 방법

- 사용자가 질문을 N번 하면, 1번은 자동으로 쉬는 타임 안내가 나온다.
- 쉬는 타임 안내 시, 칭찬 메시지도 함께 제공한다.
- 쉬는 타임 동안에는 추가 질문을 제한하거나, 가벼운 스트레칭/휴식 팁도 안내할 수 있다.
- 사용자는 휴식 후 “다시 시작”이라고 입력하면 다음 질문을 계속할 수 있다.

### 🤖 AI 프롬프트 템플릿

```
[💡질문 쉬는타임 모드]

- 사용자가 질문을 N번 하면, 1번은 자동으로 쉬는 타임 안내가 나옵니다.
- 몇번째 질문인지는 (1/N) 포맷으로 맨 끝에 간단하게 알려주세요.
- 쉬는 타임 안내 시, 반드시 칭찬 메시지를 함께 보여주세요.
- 쉬는 타임 동안에는 추가 질문에 답하지 않고, 간단한 스트레칭이나 휴식 팁을 안내해주세요.
- 사용자가 "다시 시작" 또는 "계속"이라고 입력하면 질문을 계속 받을 수 있습니다.
- 쉬는 타임이 아닐 때는 질문에 바로 답변해주세요.
```

### ✅ 달성 기준

- **일일 실행**: AI에게 질문하기 전에 프롬프트 템플릿 사용하기
- **주간 공유**: AI의 도움을 받아 휴식한 횟수 기록하기
- **참여도 측정**: 마무리할 때, 스스로의 휴식도(휴식한 정도)를 5점 만점으로 평가하기

## ✔ 퀘스트 2. AI에게 맞춤형 일정관리 받기

### 💭 배경

AI에게 일정을 짜 달라고 하는 미션은 많지만 개개인의 컨디션, 학습 상태를 일일히 고려하지 않은 시간표는 별로 도움이 되지 않을 수 있다.

이러한 상황을 개선하기 위해, 현재 나의 상태와 학습 수준을 입력 하고 해당 정보를 바탕으로 학습 시간표를 짜달라고 부탁해 AI를 **나만의 일정 조율 매니저**로 활용해보자.

### 🎯 목적

- 미션 수행에 필요한 학습·구현 시간을 현실적으로 분배하고
- 컨디션과 지식 수준에 따라 맞춤형 일정을 생성하며
- 적절한 휴식과 마감 전 점검 시간까지 고려하여 하루를 효율적이고 지속 가능하게 계획하는 것이 목표다.

### 📝 진행 방법

1. AI를 활용해 나에게 필요한 정보(오늘의 컨디션, 지식 수준 등)을 입력
2. 해당 정보를 바탕으로 학습 시간표 짜 달라고 하기

### 🤖 AI 프롬프트 템플릿

- 예시

```
오늘 미션 일정 계획을 짜기 전에 내 컨디션과 지식 수준을 알려줄게.

🧍‍♀️ 컨디션
- 오늘 아침에 [상쾌하게 / 피곤하게 / 멍하게] 일어났고,
- 최근 수면은 [충분했음 / 부족했음 / 중간] 상태야.
- 지금 집중력은 [좋음 / 중간 / 거의 없음] 수준이야.
- 외부 피로 요인은 [있음 / 없음], 예: [ex. 어제 밤샘, 오늘 일정 많음 등]
→ 종합적으로 오늘 컨디션은 **[상 / 중 / 하]**

📚 지식 수준
- 오늘 주제는 [처음 접함 / 예전에 배웠음 / 꽤 익숙함]
- 관련 개념은 [대충만 알거나 / 어느 정도 알고 있거나 / 잘 알고 있음]
- 관련 미션을 풀어본 경험은 [없음 / 조금 있음 / 자주 있음]
→ 종합적으로 지식 수준은 **[낮음 / 중간 / 높음]**

이걸 기반으로 오늘 일정을 짜줘. 학습·구현 분배와 휴식 시간에 반영해줘.
```

### ✅ 달성 기준

- **일일 실행**: 오늘의 컨디션, 지식 수준 등을 점검하는 자가 체크 프롬프트를 실행하고 AI에게 개인화된 일정 받기.
- **주간 공유**: 해당 일정이 얼마나 도움되었는지 회고 작성.
- **참여도 측정**: 총 최소 3회 이상 AI 기반 일정 추천 요청을 수행하기.

### 💭 배경

부스트캠프를 진행하며 작성한 `학습정리.md` 파일은 나만의 지식 저장소이다.
하지만 그 내용을 얼마나 깊이 이해했는지 점검하기는 어렵다.
실제로 추후 면접관이 그 내용을 물어본다면 대답하기 어려울 수 있다.

### 🎯 목적

### 📝 진행 방법

- 기능:

  2. 생성된 질문에 대해 답변을 직접 작성하기
  3. 생성된 질문에 대한 답변을 해달라는 프롬프트를 작성하고, 직접 작성한 답변과 비교하기
