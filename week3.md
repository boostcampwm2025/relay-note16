## ✔ 퀘스트 1. 세 번 말하면 한 번 쉬기

### 💭 배경

공부를 하다 보면 집중해서 연속적으로 질문을 많이 하게 된다. 하지만 너무 몰입하면 금방 지치고 효율이 떨어질 수 있다.

### 🎯 목적

50분 공부하고 10분 쉬는 것 처럼, AI에게도 쉴 시간을 주자. AI가 쉴 때 나도 쉬어서 나에게도 휴식을 주자!

### 📝 진행 방법

- 사용자가 질문을 N번 하면, 1번은 자동으로 쉬는 타임 안내가 나온다.
- 쉬는 타임 안내 시, 칭찬 메시지도 함께 제공한다.
- 쉬는 타임 동안에는 추가 질문을 제한하거나, 가벼운 스트레칭/휴식 팁도 안내할 수 있다.
- 사용자는 휴식 후 “다시 시작”이라고 입력하면 다음 질문을 계속할 수 있다.

### 🤖 AI 프롬프트 템플릿

```
[💡질문 쉬는타임 모드]

- 사용자가 질문을 N번 하면, 1번은 자동으로 쉬는 타임 안내가 나옵니다.
- 몇번째 질문인지는 (1/N) 포맷으로 맨 끝에 간단하게 알려주세요.
- 쉬는 타임 안내 시, 반드시 칭찬 메시지를 함께 보여주세요.
- 쉬는 타임 동안에는 추가 질문에 답하지 않고, 간단한 스트레칭이나 휴식 팁을 안내해주세요.
- 사용자가 "다시 시작" 또는 "계속"이라고 입력하면 질문을 계속 받을 수 있습니다.
- 쉬는 타임이 아닐 때는 질문에 바로 답변해주세요.
```

### ✅ 달성 기준

- **일일 실행**: AI에게 질문하기 전에 프롬프트 템플릿 사용하기
- **주간 공유**: AI의 도움을 받아 휴식한 횟수 기록하기
- **참여도 측정**: 마무리할 때, 스스로의 휴식도(휴식한 정도)를 5점 만점으로 평가하기

## ✔ 퀘스트 2. AI에게 일정관리 맡기기

### 💭 배경

AI에게 일정을 짜 달라고 하는 미션은 많지만 개개인의 컨디션, 학습 상태를 일일히 고려하지 않은 시간표는 별로 도움이 되지 않을 수 있다.

이러한 상황을 개선하기 위해, 현재 나의 상태와 학습 수준을 입력 하고 해당 정보를 바탕으로 학습 시간표를 짜달라고 부탁해 AI를 **나만의 일정 조율 매니저**로 활용해보자.

### 🎯 목적

- 미션 수행에 필요한 학습·구현 시간을 현실적으로 분배하고
- 컨디션과 지식 수준에 따라 맞춤형 일정을 생성하며
- 적절한 휴식과 마감 전 점검 시간까지 고려하여 하루를 효율적이고 지속 가능하게 계획하는 것이 목표다.

### 📝 진행 방법

1. AI를 활용해 나에게 필요한 정보(오늘의 컨디션, 지식 수준 등)을 입력
2. 해당 정보를 바탕으로 학습 시간표 짜 달라고 하기

### 🤖 AI 프롬프트 템플릿

```
오늘 미션 일정 계획을 짜기 전에 내 컨디션과 지식 수준을 알려줄게.

🧍‍♀️ 컨디션
- 오늘 아침에 [상쾌하게 / 피곤하게 / 멍하게] 일어났고,
- 최근 수면은 [충분했음 / 부족했음 / 중간] 상태야.
- 지금 집중력은 [좋음 / 중간 / 거의 없음] 수준이야.
- 외부 피로 요인은 [있음 / 없음], 예: [ex. 어제 밤샘, 오늘 일정 많음 등]
→ 종합적으로 오늘 컨디션은 **[상 / 중 / 하]**

📚 지식 수준
- 오늘 주제는 [처음 접함 / 예전에 배웠음 / 꽤 익숙함]
- 관련 개념은 [대충만 알거나 / 어느 정도 알고 있거나 / 잘 알고 있음]
- 관련 미션을 풀어본 경험은 [없음 / 조금 있음 / 자주 있음]
→ 종합적으로 지식 수준은 **[낮음 / 중간 / 높음]**

이걸 기반으로 오늘 일정을 짜줘. 학습·구현 분배와 휴식 시간에 반영해줘.
```

### ✅ 달성 기준

- **일일 실행**: AI로부터 개인화된 일정표 또는 시간 계획을 받아냈다.
- **주간 공유**: 해당 맞춤형 일정이 얼마나 도움되었는지에 대한 실행 피드백 또는 회고를 작성했다.
- **참여도 측정**: 총 최소 3회 이상 AI 기반 일정 추천 요청을 수행했다.

## ✔ 퀘스트 4. AI와 함께하는 기술 면접 시뮬레이션

### 💭 배경

`학습정리.md`는 나만의 지식 아카이브이자 성장의 기록이다. 하지만 그것을 **진짜 내 지식**으로 만들기 위해서는 **면접처럼 말하며 검증하는 과정**이 필요하다.

### 🎯 목적

- **면접 대비**: 순발력과 꼬리 질문 대응력을 실전처럼 훈련
- **기술 내재화**: 학습 내용을 말로 풀며 진짜로 이해하고 있는지 점검
- **의사소통 개선**: 반복되는 말버릇이나 전달력 약점 피드백 받기

### 📝 진행 방법

#### 프롬프트 1: 학습 내용 기억시키기

내가 정리한 `학습정리.md` 내용을 AI에게 전달하고, 기억시킨다.

```text
아래는 내가 정리한 학습 내용이야. 이 내용을 기억해줘. 이걸 바탕으로 기술 면접 시뮬레이션을 진행할 거야. 기억했으면 "OK"라고 대답해.
```

```text
지금부터 넌 시니어 개발자고, 나는 이 내용으로 기술 면접을 보는 지원자야.

- 기술 이해도 (5점 만점)
- 문제 해결 능력
- 의사소통 능력
- 표현력/언어습관
- 심화 대응력

또한, 내가 대답할 때 자주 쓰는 습관적 표현이나 더 좋은 말투가 있다면 그것도 함께 알려줘.
마지막에는 내가 놓친 핵심 지식이나 추가로 공부해볼만한 것도 함께 추천해줘.
```

- 면접 종료 시 AI가 평가표와 피드백 제공

## 1. 벡터 DB가 필요한 이유

전통적인 RDBMS는 정확한 조건 검색(=, <, >, LIKE 등)에 특화되어 있습니다.

하지만 **자연어**, **이미지**, **음성** 같은 데이터는 숫자나 텍스트로 단순 비교가 어렵습니다.

> 예) "사과"와 "배"는 비슷한 과일이지만 RDB에서는 비교가 어렵습니다.
>
> 벡터 DB는 이 두 단어를 **벡터 공간에 투영**하여 **거리(유사도)**로 비교할 수 있습니다.

📌 그래서 **의미 기반 검색**이나 **AI 기반 검색**이 필요한 경우 벡터 DB가 사용됩니다.

---

## 2. 벡터란 무엇인가?

> 벡터 = 여러 숫자로 이루어진 고차원 배열

예를 들어:

| 단어   | 벡터 (3차원 예시)  |
| ------ | ------------------ |
| 사과   | [0.9, 0.2, 0.4]    |
| 배     | [0.88, 0.19, 0.41] |
| 자동차 | [0.1, 0.7, 0.9]    |

벡터 간의 거리를 계산하여 **유사한 의미**를 찾습니다.

- **거리 계산 방식**: 코사인 유사도, 유클리디안 거리 등

---

## 3. 벡터 DB와 RDBMS 비교

| 항목        | RDBMS                            | 벡터 DB                                |
| ----------- | -------------------------------- | -------------------------------------- |
| 주요 기능   | 정형 데이터 검색 (`WHERE`, JOIN) | 벡터 유사도 기반 검색 (`Top-K`)        |
| 데이터 타입 | 숫자, 문자열                     | 고차원 벡터 (ex. [0.5, 0.8, 0.2, ...]) |
| 검색 방식   | 조건 기반                        | 유사도 기반 (k-NN)                     |
| 사용 목적   | 거래, 관리 시스템 등             | 추천, 유사도 검색, 의미 기반 검색      |
| 인덱싱      | B-Tree, Hash                     | FAISS, HNSW, IVF 등                    |

function cosineSimilarity(vec1, vec2) {
  const dot = vec1.reduce((sum, v, i) => sum + v * vec2[i], 0);
  const norm1 = Math.sqrt(vec1.reduce((sum, v) => sum + v * v, 0));
  const norm2 = Math.sqrt(vec2.reduce((sum, v) => sum + v * v, 0));

<h2> 퀘스트 선택 - AI에게 일정관리 맡기기 </h2>

**선택한 이유**

이전부터 AI를 활용한 시간관리를 해보고 싶었다. 부스트캠프를 하면서 가장 어려웠던 것이 시간관리였다. 이런저런 시도를 하면서 시간관리방법을 터득하고 있지만 아직도 "이렇게 하는 게 맞나"라는 생각이 떠오를 때가 있다. 그러던 중 주변 지인으로부터 AI가 시간표나 일정 같은 것을 생각보다 잘 짜준다는 말을 들었다. 그 말을 듣고난 후부터 AI를 활용한 일정관리를 시도해보고 싶었고 마침 이번 릴레이 프로젝트 퀘스트 중에 하나로 "AI에게 일정관리 맡기기"가 있어 선택해보았다. 이 퀘스트를 통해 현재 나의 일정관리 방법과 AI의 일정관리 방법을 비교하며 나의 일정관리
문제점이 무엇인 점검하는 것에 집중하고자 한다.

**느낀점**

AI의 성능과 프롬프트의 중요성에 대해 알게되는 미션이었다! 이때까지 진행했던 릴레이 프로젝트들은 AI의 활용방안에 대해 깨닫게 되는 게 많았는데 이번 미션에서는 AI의 성능이 생각보다 좋지 않다는 것을 알게 되어서 새로웠다. LLM 모델에서 프롬프트의 중요성과 AI를 어떻게 활용하냐에 따라 끌어낼수있는 결과가 천차만별이라는 것을 확실하게 알게 되었던 것 같다.

시간관리적 측면에서도 도움이 많이 되는 미션이었다. 항상 큰 단위로 일정을 관리하였는데 30분 단위로 시간표를 짜는 AI를 보면서 내가 일정을 짤 때 시간과 목표치를 잘게 쪼개어 미션을 진행했다면 성취감을 많이 느꼈을 것 같다는 생각이 들었다. 또한, 시간을 정하고 미션을 진행하니까 시간 안에 끝내야한다는 압박감에 조금 더 빠르게 미션을 진행할 수 있었던 것도 좋았다.
